# סיכום כל הפקודות (Commands) - בעברית

## סקירה כללית

בתיקיית `.cursor/commands` יש פקודות (Commands) שמארגנות תהליכי עבודה סטנדרטיים. כל פקודה היא קובץ Markdown שמתאר תהליך עבודה מסוים.

## קטגוריות הפקודות

### 1. פקודות בדיקה (Testing) - `/testing/`

#### `/testing/run-test-suite`
**מה זה עושה:**
- מריץ את כל הבדיקות (test suite)
- מנתח תוצאות ומזהה כשלונות
- בודק כיסוי בדיקות (coverage)
- אוסף מטריקות ביצועים

**חוקים שנבדקים:**
- `tests-and-validation`
- `core-python-standards`
- `error-handling-and-resilience`
- `monitoring-and-observability`
- `performance-optimization`

#### `/testing/write-targeted-tests` ⭐ **חדש!**
**מה זה עושה:**
- **מזהה קוד חדש/משונה** (מ-git changes)
- **כותב מבחנים אוטומטית** עבור כל פונקציה/קלאס חדש
- יוצר מבחנים מקיפים:
  - Happy path tests
  - Edge case tests
  - Error handling tests
  - Type validation tests
- בודק כיסוי בדיקות (coverage)
- מריץ את המבחנים החדשים ומאמת שהם עוברים

**זה בדיוק מה שחיפשת!** פקודה זו:
- ✅ מזהה קוד חדש/משונה
- ✅ כותבת מבחנים אוטומטית
- ✅ עוקבת אחרי תקנים (pytest, Arrange-Act-Assert)
- ✅ בודקת כיסוי בדיקות
- ✅ מריצה ומאמתת את המבחנים

**חוקים שנבדקים:**
- `tests-and-validation`
- `core-python-standards`
- `error-handling-and-resilience`
- `monitoring-and-observability`
- `code-review-and-collaboration`

#### `/testing/run-evaluation-suite`
**מה זה עושה:**
- מריץ בדיקות הערכה ל-LLM (Ragas, DeepEval, LangSmith)
- בודק איכות תגובות ה-LLM
- עוקב אחר עלויות (cost tracking)
- משווה עם הערכות קודמות

**חוקים שנבדקים:**
- `llm-evaluation-and-metrics`
- `llm-judge-protocol`
- `monitoring-and-observability`

#### `/testing/evaluate-with-llm-judge`
**מה זה עושה:**
- הערכה מקיפה באמצעות LLM-as-a-Judge
- בודק ביצועים, בטיחות ולוגיקה של הסוכן
- מנתח traces מ-LangSmith
- בודק אבטחה ופרטיות

**חוקים שנבדקים:**
- `llm-judge-protocol`
- `audit-protocol`
- `security-governance-and-observability`

#### `/testing/run-all-testing` (פקודת Master)
**מה זה עושה:**
- מריץ את כל פקודות הבדיקה ברצף:
  1. Write Targeted Tests (כתיבת מבחנים לקוד חדש)
  2. Test Suite (הרצת כל המבחנים)
  3. Evaluation Suite
  4. LLM Judge Evaluation
- מספק דוח מקיף עם כל התוצאות

## תשובה לשאלה שלך

### האם יש פקודה שבודקת את כל ה-RULES?

**כן!** יצרתי פקודה חדשה בשם:

### `/review/comprehensive-rules-audit` ⭐

**מה הפקודה עושה:**
1. ✅ **עוברת על כל קובץ RULE.mdc** (כל 39+ החוקים)
2. ✅ **בודקת רלוונטיות** - האם החוק רלוונטי לקוד הנוכחי
3. ✅ **מזהה פערים** - מה חסר ביישום
4. ✅ **מציעה תיקונים** - איך לתקן מה שחסר
5. ✅ **מתקנת אוטומטית** - יכולה לתקן אוטומטית (אם מאשרים)

**איך להשתמש:**
```
/review/comprehensive-rules-audit
```

**עם תיקון אוטומטי:**
```
/review/comprehensive-rules-audit and apply fixes
```

**הפקודה מספקת:**
- רשימה מלאה של כל החוקים
- מטריצת רלוונטיות (איזה חוקים חלים)
- מטריצת תאימות (מה עומד בתקן ומה לא)
- ניתוח פערים מפורט
- המלצות תיקון ספציפיות
- ציון תאימות כולל
- רשימת משימות לפי עדיפות
