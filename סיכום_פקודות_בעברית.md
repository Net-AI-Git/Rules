# סיכום כל הפקודות (Commands) - בעברית

## סקירה כללית

בתיקיית `.cursor/commands` יש פקודות (Commands) שמארגנות תהליכי עבודה סטנדרטיים. כל פקודה היא קובץ Markdown שמתאר תהליך עבודה מסוים.

## קטגוריות הפקודות

### 1. פקודות בדיקה (Testing) - `/testing/`

#### `/testing/run-test-suite`
**מה זה עושה:**
- מריץ את כל הבדיקות (test suite)
- מנתח תוצאות ומזהה כשלונות
- בודק כיסוי בדיקות (coverage)
- אוסף מטריקות ביצועים

**חוקים שנבדקים:**
- `tests-and-validation`
- `core-python-standards`
- `error-handling-and-resilience`
- `monitoring-and-observability`
- `performance-optimization`

#### `/testing/write-targeted-tests` ⭐ **חדש!**
**מה זה עושה:**
- **מזהה קוד חדש/משונה** (מ-git changes)
- **כותב מבחנים אוטומטית** עבור כל פונקציה/קלאס חדש
- יוצר מבחנים מקיפים:
  - Happy path tests
  - Edge case tests
  - Error handling tests
  - Type validation tests
- בודק כיסוי בדיקות (coverage)
- מריץ את המבחנים החדשים ומאמת שהם עוברים

**זה בדיוק מה שחיפשת!** פקודה זו:
- ✅ מזהה קוד חדש/משונה
- ✅ כותבת מבחנים אוטומטית
- ✅ עוקבת אחרי תקנים (pytest, Arrange-Act-Assert)
- ✅ בודקת כיסוי בדיקות
- ✅ מריצה ומאמתת את המבחנים

**חוקים שנבדקים:**
- `tests-and-validation`
- `core-python-standards`
- `error-handling-and-resilience`
- `monitoring-and-observability`
- `code-review-and-collaboration`

#### `/testing/run-evaluation-suite`
**מה זה עושה:**
- מריץ בדיקות הערכה ל-LLM (Ragas, DeepEval, LangSmith)
- בודק איכות תגובות ה-LLM
- עוקב אחר עלויות (cost tracking)
- משווה עם הערכות קודמות

**חוקים שנבדקים:**
- `llm-evaluation-and-metrics`
- `monitoring-and-observability`

#### `/testing/evaluate-with-llm-judge`
**מה זה עושה:**
- הערכה מקיפה באמצעות LLM-as-a-Judge
- בודק ביצועים, בטיחות ולוגיקה של הסוכן
- מנתח traces מ-LangSmith
- בודק אבטחה ופרטיות

**חוקים שנבדקים:**
- `llm-evaluation-and-metrics`
- `audit-protocol`
- `monitoring-and-observability`
- `error-handling-and-resilience`
- `performance-optimization`
- `security-governance-and-observability`

**הערה:** הרובריקה המלאה של LLM-as-a-Judge מוגדרת **רק** בקובץ הפקודה — אין חוק RULE נפרד (`llm-judge-protocol` הוסר).

### 2. פקודות אבטחה (Security) - `/security/`

החוקים התחתיים `.cursor/rules/security` כיום: `security-governance-and-observability`, `prompt-injection-prevention`. לבדיקת תאימות רחבה לכל ה-Rules (לא רק אבטחה) — `/review/final-compliance-check`.

#### `/security/analyze-audit-logs`
**מה זה עושה:**
- מנתח לוגי ביקורת (audit logs)
- מזהה אירועי אבטחה
- בודק אנומליות
- משחזר ציר זמן (timeline reconstruction)
- מתאם אירועים

**חוקים שנבדקים:**
- `security-governance-and-observability`
- `monitoring-and-observability`

#### `/security/security-audit`
**מה זה עושה:**
- סקירת אבטחה מקיפה
- מזהה פגיעויות בקוד, תשתית ותלויות
- בודק OWASP Top 10 (יישומי LLM)
- בודק אבטחת שרשרת אספקה (supply chain)
- מריץ גם `/security/analyze-audit-logs` ומשלב ממצאים
- מציע המלצות לבדיקות אבטחה

**חוקים שנבדקים:**
- `security-governance-and-observability`
- `prompt-injection-prevention`
- `configuration-and-dependency-injection`
- `monitoring-and-observability` (יישור ציפיות לוגים/תצפית)

### 3. פקודות סקירה (Review) - `/review/`

#### `/review/code-review-checklist`
**מה זה עושה:**
- סקירת קוד מקיפה עם רשימת בדיקה מובנית
- בודק איכות קוד, פונקציונליות, בדיקות, תיעוד, אבטחה
- בודק:
  - אורך פונקציות (חייב להיות פחות מ-20 שורות)
  - Type hints מלאים
  - שימוש ב-logger (לא print)
  - כיסוי בדיקות
  - אבטחה

**חוקים שנבדקים:**
- `code-review-and-collaboration`
- `core-python-standards`
- `final-review-protocol`
- `error-handling-and-resilience`
- `tests-and-validation`
- `security-governance-and-observability`

#### `/review/final-compliance-check`
**מה זה עושה:**
- בדיקת תאימות סופית לפני commit
- בודק תאימות לכל קבצי ה-governance
- בודק:
  - ארכיטקטורה (Nodes, Resilience, Config, Prompts)
  - RAG & Data (אם רלוונטי)
  - בדיקות ואימות
  - טיפול בשגיאות
  - ביצועים
  - תיעוד
  - פריסה
  - אבטחה
  - גרסאות
  - Rate limiting
  - Human-in-the-loop

**חוקים שנבדקים:**
- `final-review-protocol`
- `core-python-standards`
- `langgraph-architecture-and-nodes`
- `multi-agent-systems`
- `configuration-and-dependency-injection`
- `prompt-engineering-and-management`
- `data-schemas-and-interfaces`
- `api-interface-and-streaming`
- `performance-optimization`
- `security-governance-and-observability`
- ועוד רבים...

### 4. פקודות ניטור (Monitoring) - `/monitoring/`

#### `/monitoring/analyze-langsmith-traces`
**מה זה עושה:**
- מנתח traces מ-LangSmith
- בודק פעולות LLM, שימוש בכלים, התנהגות סוכן
- מציע אופטימיזציות עלויות
- מנתח בחירת מודלים
- אופטימיזציה של שימוש בטוקנים

**חוקים שנבדקים:**
- `monitoring-and-observability`
- `agentic-logic-and-tools`
- `cost-and-budget-management`
- `model-routing-and-selection`

#### `/monitoring/performance-analysis`
**מה זה עושה:**
- ניתוח ביצועים מקיף
- מזהה צווארי בקבוק
- מציע אופטימיזציות
- בודק תאימות ל-SLI/SLO
- מציע תכנון קיבולת

**חוקים שנבדקים:**
- `performance-optimization`
- `monitoring-and-observability`
- `core-python-standards`

#### `/monitoring/comprehensive-system-analysis`
**מה זה עושה:**
- ניתוח מערכת מקיף
- משלב נתונים מכל המקורות
- מספק תובנות הוליסטיות
- ניתוח חיזוי
- ציון בריאות מערכת

**חוקים שנבדקים:**
- כל החוקים הרלוונטיים
- `monitoring-and-observability`
- `audit-protocol`
- `llm-evaluation-and-metrics`

#### `/monitoring/run-all-monitoring` (פקודת Master)
**מה זה עושה:**
- מריץ את כל פקודות הניטור ברצף:
  1. Analyze LangSmith Traces
  2. Performance Analysis
  3. Comprehensive System Analysis
- מספק דוח ניטור מקיף
